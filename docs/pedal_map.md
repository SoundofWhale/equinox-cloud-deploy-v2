# EQUINOX 2.0 — Карта Педалей (Pedal Map) 🎸

Этот документ является живой документацией нашей модульной архитектуры. Здесь перечислены все «педали» (сервисы), их назначение и слои приложения.

---

## 1. Слой Инфраструктуры (The Pedalboard)

Центральный реестр всех сервисов. 
- **Расположение**: `c:\AI\EQUINOX\frontend\src\lib\services.ts`
- **Роль**: Регистрирует инструменты, которые компоненты могут «втыкать» в свои разъемы.

## 2. Активные Педали (Сервисы)

| Название Педали | Назначение | Расположение |
|-----------------|------------|--------------|
| **API_URL** | Относительный путь `/api/v1` (прокси через Vite) | `services.ts` |
| **Docker Mesh** | Сетевое взаимодействие между контейнерами | `docker-compose.yml` |
| **Cloudflare** | Публичный доступ через туннель | `docker run ...` |
| **AI** | Модуль общения с ИИ (Ollama/Mistral) | `services.ts` / `internal/ai` |
| **Context** | Управление изолированным пакетом задачи | `stores/context.ts` |
| **Tasks** | Управление общим списком задач | `stores/tasks.ts` |
| **Layout** | Расчет умного расположения Планет (без наложений) | `services.ts` |

## 3. Протоколы и Разъемы

- **Vite Proxy**: Перенаправляет `/api` на `http://backend:8080` внутри Docker-сети.
- **Relative Paths**: Весь фронтенд использует `/api/v1` вместо `localhost`, что позволяет работать через Cloudflare Tunnel.
- **ContextPacket**: «Входной разъем». Содержит паспорт задачи для изоляции сессии. (`interfaces.ts`)
- **ActionEvent**: «Выходной разъем». События, которые модуль выстреливает наружу вместо записи в базу. (`interfaces.ts`)

## 4. Как это чинить?

1. **Проблема с данными задачи?** Смотри `Tasks` или `Context`.
2. **ИИ не отвечает?** Проверяй педаль `AI`.
3. **Планеты накладываются?** Смотри педаль `Layout`.
4. **Не открывается сессия?** Смотри формирование `ContextPacket` на бэкенде в `tasks.go`.
